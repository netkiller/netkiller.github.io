第 39 章 从「讯飞」到「阿里」再到「火山」谁是未来人工智能赢家？

两年前我们进入AI人工智能这个赛道，与人工服务商打了快三年的交道。

我们从最初选择「讯飞」，到后台切到「阿里」，再到目前开始关注「火山」也就是豆包的技术。

最初选择讯飞是看他语音这块业务的能力，讯飞在语音领域有先发优势，在人工智能未出现之前，讯飞处于近乎垄断的地位。

人工智能未出现之前，自然语言处理是有门槛的，科大讯飞凭借在此领域的技术优势占据国内语音处理主要市场，近乎垄断，他们用人海战术，人工采集PCM，分析和处理，做到了自然语音处理的天花板，地位无法被撼动。

但这个世界瞬息万变，人工智能出现了，NLP没有门槛了，自然语言处理这块市场一夜间被重新洗牌。用人工智能技术去训练语音识别/合成/克隆/声纹，让NLP（自然语音处理）开发难度降低，一时间几乎所有云厂商都一头扎进来，这其中也包括了百度，我们也适用过他的产品，最终放弃了。百度已不入流，它都没资格让我谈，所以本篇不涉及百度这家公司。

有了人工智能，科大讯飞的技术显得过时了，人工智能只要堆算力，砸钱就能出成果，在堆算力和砸钱面前，讯飞已经尽显乏力，跟不上行业的狂飙节奏，颓势难掩。虽然现有业务还在赚钱，维持讯飞运营，但是我们看到很多讯飞生态在崩溃，咸鱼上大量讯飞二手设备，都是停止运营产品。

开发方面，技术配合度都不及阿里和火山，讯飞销售人员还有些傲慢。于是最终选择了阿里，并不是阿里比他们都好，主要还是看生态，阿里云全家桶。

当我们All In Aliyun 阿里云全家桶后，发现阿里的弊端也不少，阿里这个企业内部管理混乱，团队分散，各种产品不同团队开发，各种实现方式，重复造轮子。就 Token 认证，就有好几个入口，好几种规则。appId,AppKey,appAccessKey…… 规则混乱。代码没有整体风格，Java 包有好多，有些可以通过 Maven 依赖，有些是Jar，还有些只有源码，让你自己集成。有些代码几年未更新，版本号上还带有时间，明显就没有release。文档也十分混乱，找不到入口。

我对阿里评价是，阿里客服很好，能给你提供情绪价值，只要你问他们都能得到答复，“但不解决问题”，态度极好。阿里客服更多充当，文档查询器，因为你在网站上根本找不到，问客服他们给你发连接。

最近我们开始关注火山引擎，火山的后台干净，没有那么多服务，对业务更专注，销售联系了好几次，后面打算尝试一下火山。

讯飞，阿里，都有历史包袱，很多老服务还有在使用，火山的技术是2024开始，是最新的技术，技术团队更专注，产品基座不用兼容老系统。

但是所有公司都存在一个问题，开发人员年龄太低，代码质量无法达到国际水平。

什么是国际水平？看看 glibc, jdk, clang,rust …… 国内很多开发人员 git 都不熟练，构建工具不熟，代码能运行阶段，没有软件工程的概念。所以阿里的很多代码，都没有上到 maven 中心库。
